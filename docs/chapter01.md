## 1.1 为什么学习 C

### 1.1.1 C 的定位

C 介于「汇编」和「更现代的高级语言」之间：

| 对比 | 说明 |
|------|------|
| 相对汇编 | 可读、可移植、表达力更强 |
| 相对 Java / Python | 更接近内存与硬件，几乎没有运行时“替你管一切” |
| 相对 C++ | 核心更小；C++ 在 C 之上加了类、模板等 |

C 的典型舞台：

- 操作系统内核与驱动（Linux 内核主体是 C）
- 嵌入式与单片机固件
- 编译器、解释器、数据库、网络中间件等系统软件
- 需要极致控制内存与性能的模块

### 1.1.2 学 C 能带来什么

1. **理解计算机怎么执行程序**：栈、堆、地址、字节、编译链接  
2. **为学其它语言打底**：C#、Java、Go、Rust 等很多概念能对上号  
3. **看懂底层文档与源码**：操作系统、协议栈、开源库  

### 1.1.3 也要知道 C 的“难”

- 几乎不帮你防内存错误（悬空指针、越界、泄漏）  
- 同一段错误代码，表现可能是崩溃、偶现错乱或“暂时看起来正常”  
- 标准只规定一部分行为；其它是**未定义行为（UB）**——后面几章会反复遇到  

学习态度建议：先搞对基础，再追求技巧；**优先写清楚、可移植的代码**。

## 1.2 C 语言简史（够用即可）

| 阶段 | 大约时间 | 你需要知道的 |
|------|----------|----------------|
| 诞生 | 1970 年代 | Dennis Ritchie，Unix 上发展出来 |
| K&R C | 1978 | 《The C Programming Language》第一版语法，较老 |
| ANSI C / ISO C89 / C90 | 1989–1990 | 标准化：函数原型、标准库框架等 |
| C99 | 1999 | `//` 注释、变长数组、`long long`、更合理的整数规则等 |
| C11 / C17 | 2011 / 2018 | 多线程原子、对齐、部分修正 |
| C23 | 较新 | 持续演进，本仓库可不急于全面跟进 |

**本仓库约定**：`run.bat` 使用 `gcc -std=c11`，即以 **C11** 为准学习。

实践提示：

```bat
gcc --version
gcc -std=c11 -Wall -Wextra -o hello hello.c
```

`-Wall -Wextra`：打开更多警告，初学阶段请重视警告，常能提前发现类型/格式不匹配。

## 1.3 用计算机语言解决问题的步骤

《C Primer Plus》把编程流程概括成与写其它工程类似的循环。结合本仓库，可以这样对应：

| 步骤 | 做什么 | 在本仓库里像什么 |
|------|--------|------------------|
| 1. 明确目标 | 输入是什么、输出是什么、边界情况 | “算出白金价值”→ `platinum.c` |
| 2. 设计 | 算法、数据结构、函数怎么拆 | 先在纸上写步骤，再开写 |
| 3. 编码 | 写成 `.c` | 放进 `chapterXX/` |
| 4. 编译链接 | 生成可执行文件 | `run.bat chapter03\platinum.c` |
| 5. 运行调试 | 测样例、看警告、修正 | 终端输入、对照书上输出 |
| 6. 维护 | 改需求、修 bug、整理注释 | Git 记录你的改动 |

初学者常犯的错：**一打开编辑器就码，目标没想清楚**。哪怕只写三行，也先想好「程序要从键盘拿什么、往屏幕打什么」。

## 1.4 程序如何变成可执行文件

### 1.4.1 四件套名词

| 名称 | 例子 | 含义 |
|------|------|------|
| 源代码 | `platinum.c` | 人类写的文本 |
| 目标代码 | `platinum.o`（本仓库脚本一般不单独留） | 已编译、尚未完全链接 |
| 库 | `libmsvcrt`、CRT 等 | 别人写好、可链接进来的代码（如 `printf` 的实现） |
| 可执行文件 | `platinum.exe` | 操作系统可以直接加载运行 |

### 1.4.2 流水线

```text
  platinum.c
      │
      │  ① 预处理（preprocessor）
      │     · 展开 #include
      │     · 处理 #define
      │     · 去掉注释（实现相关）
      ▼
  翻译单元（很大一坨“纯 C”）
      │
      │  ② 编译（compile）→ ③ 汇编（assemble）
      ▼
  目标文件（机器指令 + 符号表等）
      │
      │  ④ 链接（link）
      │     · 把你的代码和标准库等拼在一起
      │     · 解析 printf 等外部符号
      ▼
  build\chapter03\platinum.exe
```

用 gcc 可以分段看（了解即可）：

```bat
gcc -E platinum.c -o platinum.i   rem 只看预处理结果
gcc -S platinum.c -o platinum.s   rem 生成汇编
gcc -c platinum.c -o platinum.o   rem 只编译不链接
gcc platinum.o -o platinum.exe    rem 链接
```

日常学习直接：

```bat
run.bat chapter03\platinum.c
```

产物路径约定：

```text
项目根/
  build/
    chapter02/
      first.exe
    chapter03/
      platinum.exe
      ...
```

### 1.4.3 头文件 ≠ 库

这是第 1 章最容易混淆的点：

| | 头文件（如 `stdio.h`） | 库（链接时加入） |
|--|------------------------|------------------|
| 内容主要是 | **声明**（告诉编译器：有个叫 `printf` 的函数长什么样） | **定义/实现**（真正的机器码） |
| 何时介入 | 预处理 / 编译阶段 | 链接阶段 |
| 写错会怎样 | 编译器报「隐式声明」「未知标识符」等 | 链接器报「找不到符号」 |

所以：

```c
#include <stdio.h>   /* 让编译器知道 printf 的原型 */
printf("hi\n");      /* 调用；实现来自标准库，链接阶段解决 */
```

尖括号 `<stdio.h>`：在系统/编译器的标准头文件目录里找。  
双引号 `"my.h"`：通常先在当前目录找（自己写的头文件以后会用到）。

## 1.5 开发环境（Windows + 本仓库）

### 1.5.1 你需要什么

1. **编辑器**：Cursor / VS Code / 记事本都行  
2. **编译器**：MinGW-w64 等提供的 `gcc`（已在 PATH 中）  
3. **终端**：CMD（推荐用本仓库 `run.bat`）

检查：

```bat
where gcc
gcc --version
```

### 1.5.2 本仓库一键编译运行

```bat
rem 在项目根目录 D:\Projects\CPrimerPlus
run.bat chapter03\platinum.c           rem 默认静默编译，成功则直接运行
run.bat chapter03\platinum.c -v        rem 显示编译/运行日志
run.bat chapter03 --all                rem 依次跑该目录下所有 .c
```

若中文提示乱码：多半是终端代码页与脚本编码不一致；在 CMD 中可先执行 `chcp 936`，或使用本仓库文档其它章节说明。

### 1.5.3 与“IDE 里点运行”的关系

无论 Code::Blocks、Dev-C++ 还是 Visual Studio，背后仍是：**编译 + 链接 + 启动进程**。本仓库把它摊开成命令行，便于对照书本概念。

## 1.6 可移植性一句话

同一份遵循标准的 C 源码，在 Windows / Linux / macOS 上：

- **源码级可移植**：改动很少或无需改动就能编译  
- **二进制不可移植**：`.exe` 不能直接拷到 Linux 上跑  

这正是 C 的目标之一：写一份源码，各平台各自编译。

平台差异仍存在（例如 `int` 宽度历史上不统一、换行符 `\r\n` vs `\n`）。以后遇到时再查，现在记住「尽量依赖标准、少写依赖特定 OS 的代码」即可。

## 1.7 常见问题（FAQ）

**Q：我只下载了 MinGW，没有装 Visual Studio，可以学吗？**  
可以。本书前几章完全够用 gcc。

**Q：`.c` 用记事本改了中文注释乱码？**  
源文件建议 UTF-8；若编译器/终端告警，可先用英文注释，或统一编码。关键是「编辑器保存编码」和「编译器读取编码」一致。

**Q：编译成功但双击 exe 窗口一闪？**  
程序跑完就退出了。用终端运行，或在 `main` 末尾加 `getchar()`（第 2 章示例里有）。

**Q：什么是“未定义行为”？**  
标准没规定「必须怎样」。编译器可以优化成任何结果。例如有符号整数溢出、格式串与参数类型不匹配。第 3 章会用示例“踩坑”加深印象。
