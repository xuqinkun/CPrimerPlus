## 2.1 第一个程序：逐行拆解

先看一个**干净**的最小示例（建议你新建一个文件亲手敲一遍）：

```c
#include <stdio.h>      /* 1. 预处理：引入标准输入输出的声明 */

int main(void)          /* 2. 程序入口；void 表示不接收参数 */
{
    printf("Hello, C!\n");  /* 3. 调用库函数输出一行文字 */
    return 0;               /* 4. 把 0 返回给操作系统，常表示成功 */
}
```

### 2.1.1 `#include <stdio.h>`

- `#` 开头的是**预处理指令**，在真正编译之前处理  
- `stdio` ≈ standard input/output  
- 没有它，旧编译器可能“猜”`printf` 的类型（危险）；现代编译器往往会直接报错或给严重警告  

### 2.1.2 `main` 函数

C 程序（可执行程序）从 `main` 开始。

常见写法：

```c
int main(void) { ... }     /* 推荐：明确无命令行参数 */
int main() { ... }         /* 在 C 里含义与 void 略有历史纠葛，入门可先当类似用法 */
int main(int argc, char *argv[]) { ... }  /* 接收命令行参数，以后再学 */
```

`main` 的返回类型是 `int`：返回给启动它的环境（在 Windows 上就是进程退出码）。

| 返回值 | 习惯含义 |
|--------|----------|
| `0` | 成功 |
| 非 `0` | 失败（具体数字可自定义约定） |

若写了 `return 0;`，与「执行到 `main` 末尾默认返回 0」（C99 起对 `main`）基本一致；初学显式写上更清晰。

### 2.1.3 花括号与语句块

```c
{
    /* 这是一个块（block） */
    int x = 1;     /* 语句以分号结束 */
    printf("%d\n", x);
}
```

- `{` `}` 圈出函数体或复合语句  
- **每条语句以 `;` 结束**（预处理指令除外）  
- C **不强制**缩进，但缩进是给人看的——请保持一致（本仓库可用 4 空格）

### 2.1.4 `printf`

```c
printf("Hello, C!\n");
```

- 第一个参数是**格式字符串**  
- `\n` 是转义字符，表示换行（见下节）  
- 带 `%d`、`%f` 等占位符时，后面还要跟对应参数（第 3 章重点）

`printf` 成功时返回写出的字符数；失败返回负值。入门阶段可不接收返回值。

## 2.2 注释

两种风格：

```c
/* 块注释：可跨多行，C89 就支持 */
// 行注释：从 // 到行尾，C99 起标准支持
```

注意：

```c
/* 不能 /* 嵌套 */ 这样写会提前结束 */
```

好注释写**意图**和**易误解之处**，不要复述代码字面意思。

## 2.3 转义字符（本章够用集合）

在字符串或字符常量里，`\` 引入特殊含义：

| 转义 | 含义 | 例子 |
|------|------|------|
| `\n` | 换行 | `"line1\nline2"` |
| `\t` | 水平制表符（Tab） | `"列1\t列2"` |
| `\\` | 反斜杠本身 | `"C:\\Users"` |
| `\"` | 双引号 | `"他说:\"你好\""` |
| `\'` | 单引号（字符常量里更常用） | `'\''` |
| `\0` | 空字符（字符串结束标记，后面章节） | — |

错误示范：想打印换行却写成 `printf("Hello/n");`——那只是字母 `/` 和 `n`。

## 2.4 函数：声明、调用、定义

程序稍复杂就会把任务拆成多个函数。关系如下：

```text
函数原型（声明） ──告诉编译器：名字、参数、返回类型
       │
       ▼
在 main（或其它函数）里调用
       │
       ▼
函数定义 ──真正的函数体（可以在调用点之前或之后）
```

### 2.4.1 为什么需要原型

若函数定义写在 `main` **后面**，又没有原型，编译器在看到调用时还不知道参数类型——这是老式 C 的坑，现代应避免。

正确模式：

```c
#include <stdio.h>

void cheer(void);          /* 原型：无参数，无返回值 */

int main(void)
{
    cheer();               /* 调用 */
    return 0;
}

void cheer(void)           /* 定义 */
{
    printf("Go!\n");
}
```

也可以把整个定义放在 `main` 前面，那就不必再单独写原型（定义本身也充当声明）。

### 2.4.2 `void` 的两种常见用法

```c
void f(void);   /* 不接受参数；调用写 f(); */
void g(int x);  /* 接受一个 int；返回类型 void 表示无返回值 */
```

不要写成：

```c
void f();       /* 在 C 中历史含义是“参数未原型化”，易踩坑；请写 void */
```

### 2.4.3 本仓库 `first.c` 对照

当前文件结构是：

```c
#include <stdio.h>

void test();           /* 原型（建议写成 void test(void);） */

int main(void)
{
    int num = 1;
    /* 下面这行是错误示范，见 2.5 */
    int n, int n2, int n3;

    printf("I am a student.\n");
    test();
    printf("First=%d\n", num);
    getchar();         /* 等待一次按键，防止部分环境下窗口闪退 */
    /* return 0; 被注释了：C99 起 main 落到末尾仍可等价于 return 0 */
}

void test()
{
    printf("test\n");
}
```

运行时大致输出：

```text
I am a student.
test
First=1
```

然后卡在 `getchar()`，按回车结束。

## 2.5 声明变量的正确写法

### 合法

```c
int num;
int a, b, c;           /* 同一类型多个变量 */
int x = 1, y = 2;      /* 声明并初始化 */
```

### 非法（`first.c` 里这种）

```c
int n, int n2, int n3; /* 错误：逗号分隔列表里不能反复写类型名 */
```

应改为：

```c
int n, n2, n3;
```

或：

```c
int n;
int n2;
int n3;
```

若你本地 `first.c` 仍是错误写法，用 `run.bat chapter02\first.c -v` 应能直接看到编译失败——这正好练习读报错。

### 命名规则（标识符）

- 可由字母、数字、下划线组成  
- **不能以数字开头**  
- 区分大小写：`Num` ≠ `num`  
- 不要与关键字冲突（`int`、`return`、`void`…）  
- 习惯：变量用小写或 snake_case（`student_count`）

## 2.6 简单用户交互预习

第 2 章书中也会提到读键盘；正式细节在第 3、4 章。这里先建立感觉：

```c
#include <stdio.h>

int main(void)
{
    int age;
    printf("Input age: ");
    scanf("%d", &age);          /* &age：把变量的地址传给 scanf */
    printf("You are %d\n", age);
    return 0;
}
```

牢记两点预习：

1. `scanf` 的格式串要与变量类型对应（`%d` ↔ `int`）  
2. 读入普通变量时，参数前通常要加 `&`  

`getchar()`：读一个字符；在 `first.c` 里用来“停住”程序。

## 2.7 程序状态：调试时脑子里要有的图

执行到 `main` 里某一行时：

```text
┌─────────────────────────────┐
│ 代码：正在执行的语句         │
├─────────────────────────────┤
│ 变量：num == 1 等            │
├─────────────────────────────┤
│ 屏幕上已打印的文字           │
└─────────────────────────────┘
```

初学调试手段：

1. 多打几个 `printf`，输出关键变量  
2. 阅读编译器**-Wall** 警告  
3. 用 `run.bat ... -v` 看完整编译命令与报错

## 2.8 常见错误与读懂报错

| 症状 | 可能原因 | 怎么处理 |
|------|----------|----------|
| `printf` was not declared / implicit declaration | 缺 `#include <stdio.h>` | 补上头文件 |
| expected ‘;’ before … | 漏分号、花括号不配对 | 从报错行往上看 |
| conficting types for ‘main’ | 两个文件都提供了 `main` 却被一起链接 | 一次只编译一个“主程序” |
| undefined reference to `xxx` | 声明了/调用了但没有定义，或没链上库 | 检查函数是否写了定义 |
| 窗口一闪而过 | 双击运行，程序瞬间结束 | 用终端或加 `getchar()` |
| 乱码 | 源文件编码与终端编码不一致 | 统一编码或少用中文输出先 |

报错信息里的 **行号**是第一线索：先打开对应行，再向上找未闭合的引号/括号。

## 2.9 编码风格建议（现在养成）

1. 一个函数做一件事，名字用动词或动词短语（`print_report`）  
2. `{` 的放置风格选一种并保持（K&R 或 Allman）  
3. 运算符两边加空格：`a = b + c;`  
4. 魔法数字先少用；以后用 `#define` 或 `const`（第 3 章）  
5. 提交到 Git 前：至少自己跑通一遍

## 2.11 练习建议

1. **默写**：不看资料写 `hello.c`，编译运行。  
2. **改错**：把 `first.c` 中非法声明改成合法，去掉不必要的警告。  
3. **加函数**：再写一个 `void goodbye(void)`，在 `main` 末尾调用。  
4. **对比**：写两个版本——函数定义在 `main` 前 / 在后（后者加原型）。  
5. **观察退出码**（可选）：在 `main` 里 `return 3;`，在 CMD 中运行后执行 `echo %ERRORLEVEL%`。
